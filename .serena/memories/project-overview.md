# DEViLBOX Project Overview

## What is DEViLBOX?

A sophisticated Amiga/Tracker music production and editing application in the browser, with:
- Native WASM playback for 175+ tracker formats
- TB-303 acid synth with Devil Fish mods
- Modular synth builder
- Hardware emulators (Gearmulator, various trackers)
- Full pattern editing across all formats
- Import/export with 1:1 integrity
- Buzzmachines effect processor
- Live parameter editing during playback

## Project Architecture

### WASM Modules (src/ directories with -wasm suffix)
Each format has its own WASM module compiled from reference C code via Emscripten:
- Hardware replayers: `klystrack-wasm/`, `gearmulator-wasm/`, `uade-wasm/`
- Tracker parsers: `hively-wasm/`, `fc-wasm/`, `jamcracker-wasm/`, `octamed-wasm/`, `musicline-wasm/`, etc.
- Hardware UIs: `pt2-sampled-wasm/` (ProTracker 2 UI), `ft2-sampled-wasm/` (FastTracker 2 UI)
- Effects: `buzzmachines/` WASM (Buzz effect units)
- DSP: `assembly/` (AssemblyScript WASM for DEViLBOX DSP engine)

**Critical:** Reference binary data (fonts, sprites) must be COPIED from Reference Code, never generated by agents.

### TypeScript Architecture
- `src/engine/` - Core engine logic (DB303Synth, tone/audio handling, format engines)
- `src/components/` - React UI (instrument editors, track view, effects)
- `src/hooks/` - State management (useTrackerStore, useFormatStore, useProjectPersistence)
- `src/pixi/` - PixiJS v8 rendering (alternative GL-based UI to React)
- `src/constants/` - Configuration, presets, defaults
- `public/` - WASM outputs, assets, test files

### Key Patterns
1. **Hardware emulation via WASM + C transpile** - Amiga replayers and other trackers
2. **Dual rendering** - React (DOM) views and PixiJS (GL) views, must be 1:1 identical
3. **Ref-based state for controls** - React knobs/sliders use `configRef` to avoid stale state bugs
4. **AudioContext + Tone.js** - Web Audio API with Tone.js abstractions
5. **Format plugins** - Each tracker format has parser, exporter, engine, UI components

## Format Coverage (175 total formats)

### Tier 1 - Full Integration (7)
HivelyTracker/AHX, Future Composer 1.3-1.4, JamCracker, MED/OctaMED, MusicLine - all have WASM engine, parser, exporter, instrument editor, and pattern views.

### Tier 2 - Partial (25)
Have WASM playback but missing exporters and/or instrument editors.

### Tier 3 - Parser Only (96)
Can import but plays via UADE fallback.

### Tier 4 - UADE Only (47)
No parser, pure UADE passthrough.

**Plan:** Bring all Tier 2+ formats to full Tier 1 integration per the transpile-debug-68k-replayer skill.

## Critical Rules & Patterns

### Git Safety (CRITICAL - Data loss incident on 2025-01-29)
- NEVER `git reset --hard`, `checkout .`, `restore .`, or `clean -f` without explicit user approval
- ALWAYS show user what changes exist first
- Safe alternatives: `git stash`, `git stash -u`, backup branches
- When in doubt, DO NOT RUN THE COMMAND

### React Knob/Control Pattern
**TB-303Controls** (`src/components/instruments/controls/JC303StyledKnobPanel.tsx`) is the reference.
```typescript
const configRef = useRef(config);
useEffect(() => { configRef.current = config; }, [config]);
const updateParameter = useCallback((key: string, value: number) => {
  onChange({ ...configRef.current, [key]: value }); // Use ref, NOT prop!
}, [onChange]); // Remove config from deps
```

### TB-303 / DB303 (Definitive Reference - See CLAUDE.md)
- WASM at `public/db303/DB303.wasm` (MD5: c618210d83ad1bbf613afb84a5adc117)
- Reference app defaults from `db303-local/default-preset.xml`
- **Critical:** TypeScript sends 0-1 normalized values. WASM does all conversions internally.
- **filterSelect:** Only 0 (DiodeLadder) or 5 (MissThang-20) valid. 1 is BROKEN.
- Inverted params: `passbandCompensation` and `resTracking` require 1-x conversion
- App state 0-1 → WASM conversion happens at `DB303Synth.ts`

### No Emoji Characters in Pixi UI
Use FontAudio icons from `PixiIcon` component, not Unicode emoji.

### UI Icon System
Use Lucide React icons for DOM components, FontAudio for Pixi.
