### Key Architecture (for continuing agent)

**Store architecture:**
- `useFormatStore` — format-specific state (klysNative, hivelyNative, jamCrackerFileData, editorMode)
- `useTrackerStore` — core tracker state (patterns, positions, instruments)
- `useEditorStore` — editor state (recordMode, editStep, currentOctave)
- `useTransportStore` — playback (isPlaying, currentRow, speed, bpm)
- `useUIStore` — modals (openModal, closeModal, modalData)

**Two engine architectures:**
1. **Standalone File Replayers** (`NativeEngineRouting`, `suppressNotes=true`) — Hively, Klystrack, JamCracker, FuturePlayer, MusicLine
2. **Per-Note Synth Triggers** (standard TrackerReplayer) — SonicArranger, SoundMon, SidMon, HippelCoSo, RobHubbard, DavidWhittaker, Fred, TFMX, FC, OctaMED, DigMug

**Pixi component APIs (common mistakes to avoid):**
- `Txt` uses `className` strings, NOT `style` objects
- `PixiModalHeader` has NO `icon` prop
- Always use `configRef` pattern for knob/slider callbacks (see CLAUDE.md)
- Never use emoji — use FontAudio icons via `PixiIcon`

**Format registration points:**
- `src/lib/import/FormatRegistry.ts` — format detection (single source of truth)
- `src/engine/InstrumentFactory.ts` — synth creation
- `src/components/tracker/TrackerView.tsx` — DOM view routing
- `src/pixi/views/PixiTrackerView.tsx` — Pixi view routing
