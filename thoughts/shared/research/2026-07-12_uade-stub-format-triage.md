# UADE stub-format triage — bucket 1 (28 lossy decode-encode stubs)

All 28 currently emit a STUB editable grid: `emptyRows` (64 fake empty rows) +
`decodeCell: decodeMODCell` (generic codec over raw bytes at offset 0). No real
pattern data exposed. Fixing byte-exact = locate the REAL contiguous on-disk
pattern region (offset+size) from the UADE eagleplayer ASM, then either decode a
real grid or apply per-byte blockRows carriers over that real region (legit —
real bytes/offsets/sizes, NOT the fabricated sizes that block futurePlayer/
steveTurner). See [[project_uade_carrier_sweep_complete]] and
docs/FORMAT_COMMAND_STREAM_GRID.md.

## EASY — parser already locates real pattern/sequence pointer tables
- **digitalSonixChrome** (DigitalSonixChromeParser.ts) — header fully decoded;
  nLengths/nSamples/seqCount read (216–218), `seqTableOff` calculated (222) but
  never read. ASM `DigitalSonixChrome_v1.asm` has layout in comments. START HERE.
- **paulRobotham** (PaulRobothamParser.ts) — header D1=voices/D2=seqs/D3=patterns;
  sequence+pattern pointer tables already located (188–189) but not decoded.
  ASM `Paul Robotham_v1.asm` / `.AMP.asm`.

## MEDIUM — spec available, real pattern region exists, needs locating
- **coreDesign** (CoreDesignParser.ts) — HUNK section-relative pointer chains
  mapped (149–156), sample extraction works; pattern table pointer needs finding.
  ASM `Core Design.asm`.
- **desire** (DesireParser.ts) — opcode-discovery framework proven for samples
  (0xE341/0x47FA chain); adjacent pointer likely → pattern table. ASM `Desire_v1.asm`.
- **glueMon** (GlueMonParser.ts) — 5-byte cells documented in parser comments
  (19–20) but never read. Player binary only (no ASM).
- **quartet** (QuartetParser.ts) — 3 variants QPA/SQT/QTS; pattern pointers
  calculated but not followed. ASM `Quartet_v1.asm` (+PSG/ST variants).
- **paulShields** (PaulShieldsParser.ts) — 3 sub-variants, sample extraction done;
  pattern structure undefined. ASM `Paul Shields_v3.asm`/`.AMP.asm`.
- **soundPlayer** (SoundPlayerParser.ts) — 15-byte header fully specified (9–17),
  IFF 8SVX sample extraction done; pattern in chunks unknown. ASM `SoundPlayer_v1.asm`.

## OPAQUE — no documented on-disk pattern stream (runtime-generated) — DEFER, do NOT drop
maniacsOfNoise, steveBarrett, sunTronic, scumm, jasonPage, markCooksey, fredGray,
jankoMrsicFlogel, seanConnolly, markII (SimpleAmigaStubParser), mikeDavies
(SimpleAmigaStubParser), jasonBrooke. Music generated by the player's 68k code at
runtime; no obvious contiguous pattern data on disk from static analysis.

**USER DIRECTIVE (2026-07-12): do NOT drop these — "if we need to emulate the
player, so be it, we have uade!"** Approach: run each player under UADE emulation
and trace which FILE bytes the replayer reads (dynamic memory-access analysis) +
how it decodes them, to locate + reverse the real on-disk pattern layout, then
build a byte-exact codec on the real bytes (NOT a fabricated grid — the emulator
reveals the true structure, so it is legitimate, not metric-gaming). Large
structural effort; scheduled after EASY+MEDIUM. Task #5.
